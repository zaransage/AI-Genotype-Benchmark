# AI Genotype Benchmark

Benchmark data and generated code from an experiment comparing two AI coding prompt strategies across three Python REST API services.

## What's here

| Path | Description |
|---|---|
| `bootstrap.sh` | Harness that ran all benchmark sessions via the Claude Code CLI |
| `baseline_<service>_<run>/` | Code generated by the baseline (free-form best practice) prompt |
| `genotype_<service>_<run>/` | Code generated by the genotype (hexagonal architecture) prompt |
| `results_*.log` | Raw timestamped output from each benchmark session |
| `results_summary.md` | Full results tables for article reference |

## Services

- **tic_tac_toe** — two-player game REST API
- **crontab_clone** — job scheduler REST API
- **ui_dashboard** — metrics dashboard REST API

## Variants

- **Baseline** — model instructed to use its own best judgment; no architectural conventions
- **Genotype** — model instructed to follow hexagonal architecture conventions from [Agentic-Code-Genotype](https://github.com/zaransage/Agentic-Code-Genotype-main)

Each variant was run 3 times per service (18 total runs). Each run had two phases: **build** (initial API) and **expand** (add web UI + SQLite persistence).

## Key findings

See `results_summary.md` for full tables. Headlines:

- Genotype builds took **~4–5× longer** and generated **~3.5× more LOC** at build time
- Genotype produced **~2.5× more test cases** initially
- Total cost per run: **~$0.74 baseline** vs **~$2.07 genotype** (2.8× multiplier)
- Model: `claude-sonnet-4-6`
